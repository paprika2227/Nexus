## Becoming a Maintainer

We're currently **not accepting** additional maintainer applications. However, if you're interested in contributing:

1. **Start Contributing:** Submit high-quality pull requests
2. **Be Active:** Help in the support server and GitHub issues
3. **Show Expertise:** Demonstrate security awareness and code quality
4. **Build Trust:** Contribute consistently over several months

When we're ready to add maintainers, we'll reach out to our most trusted contributors.

## Maintainer Guidelines

### Decision Making

**Major Decisions** (require Lead Maintainer approval):

- Breaking changes to the API or bot
- Adding/removing major features
- Monetization or premium features
- Rebranding or name changes
- Partnership agreements
- Major infrastructure changes

**Day-to-Day Decisions** (any maintainer can approve):

- Bug fixes and patches
- Documentation updates
- Minor feature improvements
- Community support
- Issue triaging

**In Case of Disagreement:**

- Discuss respectfully in private maintainer channels
- Seek consensus when possible
- Lead Maintainer has final say if consensus can't be reached

### Code of Conduct for Maintainers

All maintainers must:

- ✅ Be respectful and professional at all times
- ✅ Act in the best interests of the project and community
- ✅ Maintain confidentiality of sensitive information (see CO_MAINTAINER_AGREEMENT.md)
- ✅ Follow security best practices (2FA, secure credentials, etc.)
- ✅ Be transparent about conflicts of interest
- ❌ Never abuse access for personal gain
- ❌ Never share confidential information publicly
- ❌ Never make major decisions without proper discussion

### Confidentiality

Maintainers have access to sensitive information including:

- Internal metrics and analytics
- Strategic plans and competitor research
- User data and moderation logs
- Security vulnerabilities before public disclosure

**All maintainers must sign a confidentiality agreement** before receiving access to sensitive resources.

See [CO_MAINTAINER_AGREEMENT.md](./CO_MAINTAINER_AGREEMENT.md) for full details.

### Removing Maintainers

A maintainer may be removed for:

- **Inactivity:** No activity for 90+ days without communication
- **Breach of Trust:** Sharing confidential information
- **Misconduct:** Unprofessional behavior, harassment, or abuse
- **Malicious Actions:** Sabotage, data theft, or harmful activity
- **Conflict of Interest:** Creating competing projects with confidential info

The Lead Maintainer has sole authority to remove maintainers.
