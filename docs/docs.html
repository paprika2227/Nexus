  3. Configure /logs setup for logging
  4. Enable /antiraid config and /security config
  5. Set up /joingate setup for verification

Anti-Raid System

How it works:

  • Monitors all member joins in real-time
  • 4 detection algorithms analyze join patterns
  • Triggers when thresholds exceeded
  • Automatically bans/kicks/quarantines raiders
  • Locks down server during active raids
  • Reports threats to intelligence network

Algorithms:

  1. Rate-Based: Monitors joins per second
  2. Pattern-Based: Detects coordinated joins (similar account ages, usernames)
  3. Behavioral: Analyzes account characteristics (avatars, age, discriminators)
  4. Network-Based: Identifies account creation time clusters

Anti-Nuke System

Monitored Actions:

  • Channel deletions (triggers on 1 deletion)
  • Channel creations (triggers on 2+ in 5 seconds)
  • Role deletions (triggers on 1 deletion)
  • Role modifications (permission changes)
  • Mass bans (2+ in 5 seconds)
  • Mass kicks (2+ in 5 seconds)
  • Webhook creations (2+ in 5 seconds)
  • Permission testing (3+ changes in 10 seconds)

Response:

  • Instant ban of attacker
  • Remove all dangerous permissions
  • Trigger auto-recovery
  • Alert administrators
  • Report to threat network

Snapshot & Recovery System

Automatic Snapshots:

  • Taken every hour automatically
  • 24-hour retention (24 snapshots)
  • Captures: channels, roles, permissions, settings, webhooks, emojis, stickers
  • Zero configuration required

Recovery Process:

  1. Parallel batch processing (10 channels at once)
  2. Smart ordering (categories ‚Üí channels ‚Üí permissions)
  3. Role hierarchy preservation
  4. Verification after restoration

Commands:

  • /snapshot list - View available snapshots
  • /snapshot create - Manual snapshot
  • /snapshot restore - Restore to specific snapshot

Threat Intelligence Network

How it works:

  • When Nexus detects a threat, it's reported to the global database
  • Threat receives risk score based on severity and verification
  • All Nexus servers check new members against the database
  • Known threats are flagged/quarantined automatically

Commands:

  • /threatnet check user - Check if user is known threat
  • /threatnet stats - View network statistics

üîå Self-Hosting (Advanced)

Requirements

  • Node.js v16 or higher
  • npm or yarn
  • Discord bot token
  • Bot list API tokens (optional)

Installation

git clone https://github.com/Azzraya/Nexus.git
cd Nexus
npm install

Configuration

Create .env file:

DISCORD_TOKEN=your_bot_token
TOPGG_TOKEN=your_topgg_token (optional)
DISCORDBOTLIST_TOKEN=your_dbl_token (optional)
VOIDBOTS_TOKEN=your_voidbots_token (optional)

Running

Single Instance:

node index.js

Sharded (2,500+ servers):

node shard.js

Clustered (10,000+ servers):

npm install discord-hybrid-sharding
node cluster.js
