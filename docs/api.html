üîß Public API Documentation

Build integrations and apps on top of Nexus

üöÄ Getting Started

The Nexus Public API allows developers to integrate with Nexus, fetch bot statistics, and build custom applications.

Base URL

https://regular-puma-clearly.ngrok-free.app

üîê Authentication

Some endpoints require authentication. There are two authentication methods:

1. API Key Authentication (Recommended)

For programmatic access, use an API key in the header:

headers: {
  'x-api-key': 'your-api-key-here',
  'Content-Type': 'application/json'
}

How to get an API key:

  • Join our Discord server
  • Use /apikey generate command in any channel
  • Keep your key secret - treat it like a password!
  • Regenerate if compromised: /apikey regenerate

2. Admin Password (Dashboard Only)

For admin endpoints, use the admin password:

headers: {
  'x-admin-password': 'your-admin-password'
}

üîí Security Best Practices

  • Never expose API keys in client-side code
  • Use environment variables for API keys
  • Rotate keys periodically
  • Use different keys for dev/production
  • Monitor API usage for suspicious activity

üéØ Which Endpoints Require Auth?

No Auth Required:

  • GET /api/v1/stats
  • GET /api/v1/version
  • GET /api/v1/server/:id/health
  • GET /api/v1/user/:id/risk
  • POST /api/v1/feedback
  • POST /api/v1/showcase/nominate

Requires API Key:

  • All moderation endpoints (ban, kick, warn, bulk)
  • Server configuration endpoints
  • Backup/restore endpoints
  • Custom command management
  • Data export endpoints
  • Webhook management
  • Workflow creation

üìä Rate Limits

All public endpoints are rate-limited to 100 requests per minute per IP address.

100 req/min

Authenticated requests get higher limits: 500 requests per minute.

üì¶ Response Format

All responses are in JSON format with appropriate HTTP status codes.

üîî Webhooks

        
Receive real-time events from Nexus delivered to your server.


        
How Webhooks Work

        

          Nexus can send HTTP POST requests to your server when events occur:
        

        
    
          
  • User banned or kicked
    • 
          
  • Security threat detected
    • 
          
  • Server joins/leaves
    • 
          
  • Custom command executed
    • 
          
  • Workflow triggered
    • 
        


        
Setting Up a Webhook

        

          
// Register your webhook endpoint
fetch('https://regular-puma-clearly.ngrok-free.app/api/v1/webhooks/create', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': 'your-api-key',
    'ngrok-skip-browser-warning': 'true'
  },
  body: JSON.stringify({
    serverId: '1234567890',
    url: 'https://your-server.com/webhook',
    events: ['user.banned', 'threat.detected', 'server.joined'],
    name: 'My Webhook'
  })
});

        


        
Webhook Event Format

        
Events are sent as JSON POST requests:

        

          
{
  "event": "user.banned",
  "timestamp": 1733270400000,
  "serverId": "1234567890",
  "data": {
    "userId": "0987654321",
    "reason": "Spam",
    "moderator": "1122334455"
  },
  "signature": "sha256_hash_for_verification"
}

        


        
Securing Your Webhooks

        
    
          
  • Verify the signature header to ensure events are from Nexus
    • 
          
  • Use HTTPS endpoints only
    • 
          
  • Respond with 200 OK quickly (process async)
    • 
          
  • Handle retries gracefully
    •

‚ö° Performance Tips

        
Optimize your API usage for maximum speed and efficiency.


        
1. Use Caching

        

          
// Cache stats for 5 minutes
const CACHE_TIME = 5 * 60 * 1000;
let cachedStats = null;
let lastFetch = 0;

async function getStats() {
  if (Date.now() - lastFetch < CACHE_TIME) {
    return cachedStats;
  }
  
  cachedStats = await fetch('https://regular-puma-clearly.ngrok-free.app/api/v1/stats')
    .then(r => r.json());
  lastFetch = Date.now();
  return cachedStats;
}

        


        
2. Batch Requests

        
Use bulk endpoints instead of individual calls:

        

          
// ‚ùå BAD: Multiple individual requests
for (const userId of userIds) {
  await banUser(userId);
}

// ‚úÖ GOOD: Single bulk request
await fetch('https://regular-puma-clearly.ngrok-free.app/api/v1/moderation/bulk', {
  method: 'POST',
  body: JSON.stringify({
    action: 'ban',
    userIds: userIds
  })
});

        


        
3. Use Pagination

        
Don't fetch all data at once - paginate large result sets.


        
4. Implement Retry Logic

        

          
async function fetchWithRetry(url, options, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const response = await fetch(url, options);
      if (response.ok) return response;
    } catch (error) {
      if (i === retries - 1) throw error;
      await new Promise(r => setTimeout(r, 1000 * Math.pow(2, i))); // Exponential backoff
    }
  }
}

        


        
5. Monitor Rate Limits

        
Check response headers for rate limit info:

        

          
const response = await fetch(url);
console.log('Remaining:', response.headers.get('X-RateLimit-Remaining'));
console.log('Reset:', response.headers.get('X-RateLimit-Reset'));

        

      


      
      

        
üîç Troubleshooting

        
Common issues and how to fix them.


        
‚ùå 401 Unauthorized

        
Problem: Invalid or missing API key

        
Solution:

        
    
          
  • Verify your API key is correct
    • 
          
  • Check you're using x-api-key header
    • 
          
  • 
            Regenerate key if compromised: /apikey regenerate
          
    • 
        


        
‚ùå 429 Too Many Requests

        
Problem: Rate limit exceeded

        
Solution:

        
    
          
  • Wait before retrying (check Retry-After header)
    • 
          
  • Implement exponential backoff
    • 
          
  • Use authenticated requests for higher limits
    • 
          
  • Cache responses when possible
    • 
        


        
‚ùå 404 Not Found

        

          Problem: Endpoint doesn't exist or resource not found
        

        
Solution:

        
    
          
  • Check the endpoint URL for typos
    • 
          
  • Verify the resource (server/user) exists
    • 
          
  • Ensure you're using the correct base URL
    • 
        


        
‚ùå CORS Errors

        
Problem: Browser blocking cross-origin requests

        
Solution:

        
    
          
  • Add 'ngrok-skip-browser-warning': 'true' header
    • 
          
  • Contact us to whitelist your domain
    • 
          
  • Use server-side requests instead of browser
    • 
        


        
‚ùå Slow Response Times

        
Problem: API taking too long to respond

        
Solution:

        
    
          
  • Implement caching for frequently accessed data
    • 
          
  • Use pagination for large datasets
    • 
          
  • Check your network connection
    • 
          
  • Monitor API status page
    •
