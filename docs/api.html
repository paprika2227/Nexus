üöÄ Getting Started

The Nexus Public API allows developers to integrate with Nexus, fetch bot statistics, and build custom applications.

Base URL

https://regular-puma-clearly.ngrok-free.app

Rate Limits

All public endpoints are rate-limited to 100 requests per minute per IP address.

100 req/min

Response Format

All responses are in JSON format with appropriate HTTP status codes.

üìä Public Endpoints

GET /api/v1/stats

Get basic bot statistics including server count, user count, and uptime.

Response Example:

{
  "serverCount": 17,
  "userCount": 356,
  "avgResponseTime": 50,
  "uptime": 86400,
  "commandCount": 88
}

Example Request:

fetch('https://regular-puma-clearly.ngrok-free.app/api/v1/stats', {
  headers: {
    'ngrok-skip-browser-warning': 'true'
  }
})
.then(res => res.json())
.then(data => console.log(data));
GET /api/v1/version

Get current bot and API version information.

Response Example:

{
  "botVersion": "3.0.0",
  "apiVersion": "1.0.0",
  "botName": "Nexus",
  "uptime": 86400,
  "endpoints": {
    "server": "/api/v1/server/:id",
    "stats": "/api/v1/stats"
  }
}
GET /api/v1/recent-activity

Get recent bot activity across all servers (last 10 actions).

Response Example:

}

üí° Usage Examples

JavaScript (Fetch API)

// Get bot stats
const response = await fetch('https://regular-puma-clearly.ngrok-free.app/api/v1/stats', {
  headers: {
    'ngrok-skip-browser-warning': 'true'
  }
});
const stats = await response.json();
console.log(`Nexus is in ${stats.serverCount} servers!`);

Python (requests)

import requests

response = requests.get('https://regular-puma-clearly.ngrok-free.app/api/v1/stats')
stats = response.json()
print(f"Servers: {stats['serverCount']}")

Node.js (axios)

const axios = require('axios');

const { data } = await axios.get('https://regular-puma-clearly.ngrok-free.app/api/v1/stats');
console.log(`Server Count: ${data.serverCount}`);

Admin Request Example

// Admin endpoints require password header
fetch('https://regular-puma-clearly.ngrok-free.app/api/admin/command-analytics', {
  headers: {
    'x-admin-password': 'your-admin-password',
    'ngrok-skip-browser-warning': 'true'
  }
})
.then(res => res.json())
.then(data => console.log(data));

‚ö†Ô∏è Error Handling

The API uses standard HTTP status codes:

Status Code Meaning
200 Success
400 Bad Request - Invalid parameters
401 Unauthorized - Missing or invalid admin password
404 Not Found - Endpoint or resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Error Response Format:

{
  "error": "Error message description"
}

‚è±Ô∏è Rate Limiting

Public endpoints: 100 requests per minute per IP

Admin endpoints: No rate limit (password protected)

Rate Limit Headers:

X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1733270400000

When rate limited, you'll receive a 429 Too Many Requests response.

üîî Webhook Events

Configure webhooks in Discord using /webhook add to receive real-time events.

Available Events:

  • member.join - Member joined server
  • member.leave - Member left server
  • member.ban - Member was banned
  • member.kick - Member was kicked
  • raid.detected - Raid attempt detected
  • nuke.detected - Nuke attempt detected
  • threat.high - High threat level
  • server.health.critical - Server health critical
  • command.executed - Command was used
  • config.changed - Configuration updated

Webhook Payload Example:

{
  "event": "raid.detected",
  "guildId": "1234567890",
  "timestamp": 1733270400000,
  "data": {
    "joinCount": 15,
    "threatScore": 85,
    "patterns": ["massJoin", "newAccounts"]
  }
}
